# BRMSTE multi-cloud domain registry

**BRMSTE LTD · Companies House 15310393 · GB2607860 · PCT/GB2026/050406**

This directory is the single source of truth for the BRMSTE apex-domain fleet across
**Cloudflare · Hetzner · AWS · Azure · Siemens IEM**, and a credential-free auditor that
grades every apex against one HTTPS security policy.

## Files

| File | Purpose |
|------|---------|
| `registry.json` | Declarative registry: known apexes, per-domain cloud lane bindings, and the shared header policy. Schema `brmste-domain-registry/v2`. |
| `registry.schema.json` | JSON Schema (draft-07) for `registry.json`. |
| `polish-report.json` | Latest live audit output (generated — regenerated by the auditor / CI). |
| `POLISH.md` | Human-readable audit snapshot + **exact remediation** (Caddy header blocks, DNS publish list). |

## Scripts (repo root `scripts/`)

| Script | What it does | Credentials |
|--------|--------------|-------------|
| `verify-domains-registry.sh` | Structural gate: ids contiguous, cloud lanes complete, CF account matches `coming-soon/wrangler.toml`, Hetzner bindings resolve to `data/hetzner/servers.json`. | none (offline) |
| `polish-domains.sh` | Live auditor: DNS + public HTTPS only. Grades each apex against `policy.required_headers` / `forbidden_headers` and writes `polish-report.json`. | **none** — no tokens, no MCP, no signing |

```bash
bash scripts/verify-domains-registry.sh      # structural gate (CI + local)
bash scripts/polish-domains.sh --timeout 12  # live audit -> domains/polish-report.json
```

## Policy

Defined once in `registry.json .policy`:

- **HTTPS-only**, target HSTS `max-age=63072000; includeSubDomains; preload` (2 years, preload-eligible).
- **Required headers:** `Strict-Transport-Security`, `X-Content-Type-Options`, `X-Frame-Options`,
  `Referrer-Policy`, `Permissions-Policy`, `Content-Security-Policy`.
- **Forbidden (leak) headers:** `X-Powered-By`, `X-Middleware-Rewrite`, `X-AspNet-Version`,
  `X-AspNetMvc-Version`, `X-Runtime`.

The canonical header values live in [`coming-soon/src/index.js`](../coming-soon/src/index.js)
(`SECURITY_HEADERS`). As of 2026-07-18 the BRMSTE apexes are served by the **Hetzner Caddy
`domains-platform`** (origin `brmste-db`, 178.104.79.112) with Cloudflare in DNS-only mode, so
headers must be applied at the **Caddy layer** — see `POLISH.md`.

## Remediation taxonomy

| Action | Trigger |
|--------|---------|
| `publish-dns` | Apex has no A/AAAA record (NXDOMAIN). |
| `investigate-origin` | Resolves but no HTTP response, or HTTP ≥ 400. |
| `harden-headers` | Reached, but a required header is missing or HSTS is below target. |
| `strip-meta-headers` | Reached, but a forbidden/leak header is present. |
| `none` | Polished (all required present, HSTS at target, no leaks) or external-managed. |
